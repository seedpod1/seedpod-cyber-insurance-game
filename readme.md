# SeedPod Cyber Coverage: The Game

An educational game teaching critical cyber insurance policy concepts through interactive gameplay.

![SeedPod Cyber Logo](seedpod-cyber-logo.png)

## About

Navigate five policy lanes while answering coverage questions to master the hidden pitfalls in cyber insurance policies. This game was created by SeedPod Cyber to help insurance professionals, MSPs, and risk managers understand the nuances that can make or break a cyber insurance claim.

## What You'll Learn

- **Definitions** - How narrow policy terms can deny claims
- **Sublimits** - Understanding hard caps per coverage category
- **Exclusions** - Coverage killers that void protection
- **Vendor Coverage** - When Contingent Business Income is critical
- **Response Requirements** - Why 24-72 hour notification windows matter

## How to Play

### Desktop
- **Arrow Keys** or **WASD** to move
- **Spacebar** to pause
- Navigate through policy hazards and answer coverage questions correctly to advance

### Mobile
- **Swipe** in any direction to move
- Use on-screen buttons for controls

### Game Mechanics

1. **Navigate 5 lanes** representing different policy sections
2. **Answer coverage questions** when you encounter hazards
3. **Earn coins** for correct answers (2-3 coins per question based on level)
4. **Use the shop** to buy coverage enhancements that protect you from wrong answers
5. **Complete 5 levels** to master cyber insurance coverage

**Scoring:**
- Correct answers: +100 points each
- Coins collected: +10 points each
- Lives remaining: +500 points each
- Penalties: -50 points each

## Usage

### Running the Game

Simply open `seedpod-cyber-insurance-game.html` in any modern web browser. No installation or server required.

### Embedding

You can embed this game in your website, LMS, or training materials:

```html
<iframe src="seedpod-cyber-insurance-game.html" width="1200" height="900" frameborder="0"></iframe>
```

### Customization

The game is built with vanilla HTML, CSS, and JavaScript. You can easily customize:
- Questions and answers in the `QUIZ` object
- Shop items and costs in the `SHOP` array
- Difficulty scaling by adjusting `makeHazards()` function
- Visual styling in the `<style>` section
